### JIRA Markdown Compatibility Notes

**Note:** JIRA's native format is Atlassian Document Format (ADF). The `mcp__atlassian__addCommentToJiraIssue` MCP tool's handling of markdown-to-ADF conversion must be verified during implementation testing.

**Potential format differences to test:**

- Headers: Standard markdown `#` vs JIRA `h1.` syntax
- Bold: Standard markdown `**bold**` vs JIRA `*bold*` syntax
- Links: Standard markdown `[text](url)` vs JIRA `[text|url]` syntax
- Tables: Standard markdown vs JIRA table syntax `||Header||` and `|Cell|`

**Testing Requirement:** Verify MCP tool handles markdown conversion correctly, or implement manual conversion to ADF format if needed.

### Error Handling

**Permission Denied:**

- Display: "‚ùå Cannot add comment to AOD-1234. Check JIRA permissions."
- Action: Guide user to verify Atlassian MCP has comment permissions

**Comment Too Large:**

- JIRA API has comment size limits (verify during implementation)
- Mitigation: If comment rejected due to size, truncate sections with "Full report available in attached file"
- Fallback: Save enrichment to local file, add summary comment only with link to full enrichment

**Network Error:**

- Display: "‚ùå Cannot post comment. Check network connection."
- Action: Save enrichment locally, retry later

**Rate Limit Exceeded:**

- Display: "‚ö†Ô∏è JIRA API rate limit exceeded. Retrying in 60 seconds..."
- Action: Implement exponential backoff retry strategy (3 attempts: 60s, 120s, 240s)
- Fallback: After 3 failed retries, save enrichment locally and notify user
- User Guidance: "Comment will be posted when rate limit resets. Enrichment saved to: {local_file_path}"
